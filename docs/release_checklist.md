# SilverLink 출시 전 기능 체크리스트

## 문서 목적
이 문서는 SilverLink MVP의 출시 전 필수 기능을 점검하기 위한 체크리스트다.

현재 단계의 목표는 신규 기능 추가가 아니라,
기존 핵심 기능이 실제 사용자 흐름 안에서 안정적으로 동작하는지 확인하고,
관계 모델, UX, 정책 문구, 기본 디자인 품질을 정리하는 것이다.

---

## 1. 출시 전 최우선 원칙

- 신규 기능 추가보다 안정화가 우선이다
- 핵심 사용자 흐름이 끝까지 이어져야 한다
- 관계 모델(관심 / 1촌 / 대화)이 사용자 입장에서 이해 가능해야 한다
- 시니어 UX 기준으로 글자, 버튼, 구조가 충분히 쉬워야 한다
- 공개범위, 차단, 신고, 대화 조건이 정책과 실제 동작 모두 일치해야 한다

---

## 2. 핵심 사용자 흐름 체크

### 흐름 A
회원가입 → 온보딩 → 홈 → 사람 상세 → 관심 보내기 → 대화

### 흐름 B
회원가입 → 온보딩 → 홈 → 피드 보기 → 반응 → 프로필 → 관계 연결

### 흐름 C
회원가입 → 온보딩 → 홈 → 모임 → 참여 → 모임 채팅 → 후속 관계

### 체크 기준
- 각 단계가 끊기지 않고 이어지는가
- 다음 행동이 명확한가
- 에러가 발생하지 않는가
- 사용자 입장에서 왜 이 화면이 필요한지 이해 가능한가

## 4. 관계 모델 최종 확인 항목

아래 개념이 사용자 입장에서 분명해야 한다.

- 관심: 가벼운 신호
- 1촌: 상호 승인된 관계
- 대화: 상호 관심, 1촌, 같은 모임 참여 시 가능

### 확인 질문
- [ ] 관심과 1촌의 차이가 명확한가
- [ ] 상호 관심이면 바로 대화 가능한가
- [ ] 같은 모임 참여자는 어떤 범위까지 소통 가능한가
- [ ] 1촌이라는 용어가 직관적인가

---

## 5. 홈 화면 우선순위 확인

홈 화면은 기능 링크 모음이 아니라 첫 행동 유도 화면이어야 한다.

### 추천 우선순위
1. 추천 사람
2. 최근 공개 피드
3. 추천 모임
4. 알림 요약

### 확인 질문
- [ ] 홈의 1순위 행동이 명확한가
- [ ] 추천 사람 영역이 충분히 강조되는가
- [ ] 최근 피드가 생활형 분위기를 보여주는가
- [ ] 모임이 너무 뒤로 밀리거나 과하게 앞서지 않는가

---

## 6. 정책 문구 확인

아래 용어는 반드시 검토한다.

- 관심 보내기
- 관심 취소
- 1촌 맺기
- 대화 가능
- 공개범위
- 인증 완료
- 모집중 / 참여중 / 나가기
- 신고 / 차단

### 확인 질문
- [ ] 시니어가 이해하기 쉬운가
- [ ] 너무 기술적이지 않은가
- [ ] 너무 가볍거나 소개팅앱처럼 보이지 않는가
- [ ] 관계 서비스의 신뢰감을 해치지 않는가

## 8. 최종 목표

출시 전 목표는 기능 수를 늘리는 것이 아니라,
아래를 만족하는 것이다.

1. 핵심 사용자 흐름이 끝까지 이어진다
2. 관계 모델이 단순하고 이해 가능하다
3. 홈에서 첫 행동이 보인다
4. 시니어 UX 기준에서 어렵지 않다
5. 공개범위 / 차단 / 대화 정책이 실제 동작과 일치한다
