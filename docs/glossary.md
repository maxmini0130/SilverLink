# SilverLink 서비스 용어집

- 목적: 화면 문구, 코드 상수, 안내 문구에서 같은 개념을 같은 말로 부르기 위한 기준
- 기준일: 2026-04-16
- 관련 문서: [relationship_model.md](./relationship_model.md), [priority_plan.md](./priority_plan.md)

---

## 1. 원칙

1. **한 개념 = 한 단어**. "친구"와 "1촌"을 섞어 쓰지 않는다. 이 서비스의 단어는 **1촌**이다.
2. **동사는 행동 중심**. "관심 보내기", "1촌 맺기", "대화 시작"처럼 사용자가 누를 버튼에 맞춘다.
3. **상태는 상태답게**. "보낸 관심", "받은 관심", "상호 관심", "1촌"은 목록/뱃지에서 쓰는 상태 명사다.
4. **공개범위는 5단계 고정**. 새 단계를 추가하지 않는다. 문구는 항상 `…만 보기` 형태.
5. **시니어 관점**. 영문 약어를 쓰지 않는다 (DM → "대화", feed → "피드"는 예외로 유지).

## 3. 공개범위 용어 (표준)

공개범위 값은 아래 5개로 고정한다. 라벨은 항상 `…만 보기` 또는 `전체 …보기` 형태로 끝난다.

| 코드 값 | 라벨 (표준) | 짧은 설명 |
|--------|-----------|---------|
| `private` | **나만 보기** | 나 혼자만 볼 수 있어요 |
| `friends` | **1촌만 보기** | 1촌 관계인 분들에게만 보여요 |
| `interested` | **관심 있는 사람만 보기** | 서로 관심을 주고받은 분들에게만 보여요 |
| `same_group` | **같은 모임만 보기** | 같은 모임에 속한 분들에게만 보여요 |
| `members` | **전체 회원 보기** | 가입한 모든 회원에게 보여요 |

### 금지되는 변형 표기
- ❌ `1촌만`, `관심 있는 사람만`, `같은 모임`, `같은 모임 사람만 보기`, `전체 인증회원`, `전체 인증회원 보기`
- ✅ 항상 위 표의 "라벨 (표준)" 값만 사용

### 단일 출처
공개범위 라벨/설명은 [`web/src/lib/visibility.ts`](../web/src/lib/visibility.ts)에서만 정의한다. 각 페이지는 이 모듈에서 `VISIBILITY_OPTIONS`, `visibilityLabel()`을 가져다 써야 한다.

## 5. 안내 문구 표준 (시니어 친화)

### 관심/1촌 안내
- 관심 → 대화 유도: **"서로 관심을 보내거나, 같은 모임에 참여하면 대화를 시작할 수 있어요."**
- 상호 관심 뱃지: **"서로 관심을 보낸 사이예요"**
- 1촌 뱃지: **"1촌 사이예요"**
- 받은 관심 안내: **"나에게 관심을 보낸 분이에요"**

### 공개범위 안내
- 피드 작성 시: **"누구에게 보여줄지 선택하세요."**
- 기본 공개범위 설정: **"새 피드를 작성할 때 기본으로 선택될 공개범위예요."**

### 빈 상태 문구
- 보낸 관심 없음: **"아직 보낸 관심이 없어요. 사람 목록에서 관심을 보내보세요."**
- 받은 관심 없음: **"아직 받은 관심이 없어요."**
- 1촌 없음: **"아직 1촌이 없어요. 상호 관심이 생기면 1촌을 맺어보세요."**
- 진행 중 대화 없음: **"아직 진행 중인 대화가 없어요."**
