# SilverLink MVP

## 1. 서비스 개요

### 서비스명
SilverLink (실버링크)

### 한 줄 정의
시니어가 일상 사진과 취미를 공유하고, 공개 범위를 직접 조절하며, 비슷한 사람과 안전하게 연결되는 미니홈피형 관계 서비스

### 서비스 성격
- 노골적인 소개팅 앱이 아님
- 댓글 중심 SNS가 아님
- 불특정 다수에게 완전 공개되는 공개형 SNS가 아님
- 친구, 말벗, 동행, 취미 기반 관계가 자연스럽게 이어지는 구조를 지향
- 싸이월드식 공개범위 감성과 인스타식 피드 일부를 결합한 구조

---

## 2. 핵심 사용자

### 주요 타깃
- 50대 후반 ~ 70대 초반 시니어
- 스마트폰 사용은 가능하지만 복잡한 UX는 부담스러울 수 있는 사용자
- 노골적인 소개팅보다 자연스러운 관계 형성을 선호하는 사용자
- 친구, 말벗, 동행, 취미 기반 관계를 원하는 사용자

### 사용자 특성
- 사진, 피드, 좋아요, DM 개념에 점점 익숙해지는 세대
- 다만 공개 범위와 안전에 더 민감함
- 댓글/공개 평가 구조보다는 부드럽고 통제 가능한 관계 형성 구조를 선호할 가능성이 높음

---

## 3. 핵심 가치 제안

1. 안전한 관계 형성
2. 시니어 친화 UX
3. 공개 범위를 직접 조절할 수 있는 생활 피드
4. 온라인 관계가 오프라인 모임으로 이어지는 구조
5. 대놓고 소개팅처럼 보이지 않는 자연스러운 관계 서비스

---

## 4. 제품 원칙

- 버튼은 크게
- 글자는 크게
- 한 화면의 선택지는 적게
- 카드형 UI 우선
- 댓글 기능 없음
- 기본 공개 범위는 제한 공개
- 아무나 DM 불가
- 신고/차단은 항상 쉬워야 함
- 외부 웹 완전 공개형 구조 금지

## 6. 공개범위 정책

실버링크의 핵심 차별점 중 하나는 게시물 공개 범위 제어다.

### 공개 범위 옵션
- 나만 보기
- 1촌만 보기
- 관심 있는 사람만 보기
- 같은 모임 사람만 보기
- 전체 인증회원 보기

### 정책 원칙
- 외부 웹 검색 노출 없음
- 인증 회원 중심의 제한형 커뮤니티
- 공개 범위는 프론트 표시 수준이 아니라 데이터 접근 정책으로 제어
- 차단 사용자 간에는 조회 및 대화 제한

## 8. 핵심 사용자 흐름

### 흐름 A: 사람 중심
회원가입 → 프로필 생성 → 홈 추천 사람 → 프로필 상세 → 관심 보내기 → 상호 관심 → 대화

### 흐름 B: 피드 중심
홈 피드 보기 → 반응 보내기 → 상대 프로필 확인 → 관심 보내기 → 대화 가능 상태

### 흐름 C: 모임 중심
모임 탐색 → 모임 상세 → 참여 신청 → 모임 대화방 → 오프라인 참여 → 후속 관계 연결

## 12. 기술 아키텍처 방향

### MVP 아키텍처
- Next.js가 프론트엔드 및 기본 서버 레이어 역할
- Supabase가 Auth / Postgres / Storage 담당
- 공개범위 및 권한은 RLS 중심으로 제어
- 중요한 액션은 Next.js Server Action 또는 Route Handler로 처리

### 핵심 원칙
- 단순 조회는 비교적 빠르게 구현
- 관계 생성, 대화 가능 여부, 신고/차단 등 핵심 로직은 서버 레이어로 점진 이동
- 프론트에 비즈니스 규칙을 과하게 넣지 않음

## 16. 스택에 대한 실무 판단

### 그대로 가져가도 되는 부분
- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- React Hook Form
- Zod
- Zustand
- Vercel

이 구성은 MVP 이후에도 계속 가져가기 적합하다.

### Supabase에 대한 판단
Supabase는 MVP용 임시 선택이 아니라, 초기 운영 단계까지 충분히 사용할 수 있다.
다만 아래 원칙이 중요하다.

- RLS를 초반부터 제대로 설계
- 공개범위/권한을 DB 정책으로 제어
- 프론트가 DB를 무분별하게 직접 다루지 않도록 주의
- 관계/대화/신고 같은 핵심 로직은 서버 레이어로 점진 이동

### Prisma에 대한 판단
초기에는 굳이 넣지 않는다.
복잡한 서버 로직, 백오피스, 배치 작업, 운영툴 필요성이 커질 때 검토한다.

## 17. 한 줄 결론

실버링크 MVP는
“사진 기반 생활 피드 + 공개범위 제어 + 안전한 관계 연결 + 모임 중심 오프라인 전환”
구조로 설계한다.

초기 기술 스택은
Next.js + TypeScript + Tailwind + shadcn/ui + Supabase
구성이 가장 현실적이며, MVP 이후에도 대부분 그대로 가져갈 수 있다.
