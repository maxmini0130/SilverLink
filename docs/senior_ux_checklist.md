# SilverLink 시니어 UX 체크리스트

- 목적: 실제 UI가 60+ 사용자에게 너무 작거나 복잡하지 않은지 기계적으로 점검할 수 있는 기준
- 기준일: 2026-04-16
- 관련: [trd.md §7](./trd.md), [priority_plan.md P1](./priority_plan.md), [글꼴/버튼 전역 스타일: globals.css](../web/src/app/globals.css)

---

## 1. 원칙 (한 줄)

1. **작지 않게**: 본문 ≥ 16px, 설명 텍스트 ≥ 14px, 버튼 높이 ≥ 48px
2. **명확하게**: 아이콘 옆에 항상 텍스트 라벨
3. **친절하게**: 상태 변화는 즉시 말로 알려주기
4. **단순하게**: 한 화면 한 주행동, 그 밖엔 모두 보조

---

## 2. 전역 기준

| 항목 | 기준 | 현재 구현 상태 |
|------|-----|-------------|
| 기본 본문 크기 | ≥ 18px | ✅ `body { font-size: 18px }` |
| 입력 필드 글자 | ≥ 18px | ✅ `input, select, textarea` 18px 강제 |
| 버튼 최소 높이 | ≥ 48px | ✅ `button { min-height: 48px }` |
| 터치 타겟 | ≥ 44×44 | ⚠️ Link 태그는 패딩 직접 확인 필요 |
| 줄 간격 | ≥ 1.5 | ✅ `line-height: 1.6` |
| 색상 대비 | WCAG AA | ⚠️ 테스트 도구로 검증 필요 |

## 3. 화면 요소별 체크리스트

### 3.1 텍스트 크기

- [ ] `text-xs` (12px) 사용은 **메타데이터(날짜, 작성자 보조)에만** 허용
- [ ] 본문 설명/안내 문구는 `text-sm` (14px) 이상
- [ ] 버튼 레이블은 `text-base` (16px) 이상
- [ ] 섹션 헤더는 `text-xl` (20px) 이상

**현재 코드 점검**: `text-xs` 사용 35개소 → 모두 메타데이터 표시 여부 확인 필요 ([grep 결과](#text-xs-audit))

### 3.2 버튼 · 링크

- [ ] `min-height: 48px` 기본 적용되는가 (전역 CSS로 강제됨)
- [ ] 아이콘 버튼에 `aria-label` 또는 옆 텍스트 존재
- [ ] `disabled` 상태 색상이 구별되는가
- [ ] 처리 중(loading) 상태 표시 ("처리 중...")
- [ ] 파괴적 액션(취소, 삭제)은 확인 대화상자 또는 구분 색상 사용

### 3.3 입력 필드

- [ ] `label` 제공 (스크린리더 대응)
- [ ] `placeholder`는 예시만, 필수 안내는 `label` 또는 바로 아래 텍스트로
- [ ] 에러 메시지는 필드 바로 아래 빨간 글씨로
- [ ] 자동완성/대문자 자동화 등이 시니어에 방해되지 않는가

### 3.4 정보 밀도

- [ ] 한 카드에 정보 4개 이하 (제목, 부제, 메타, 액션)
- [ ] 리스트 아이템 간격 ≥ 12px
- [ ] 텍스트 덩어리 80자 이내로 줄바꿈
- [ ] 작은 아이콘이 핵심 정보 전달에 쓰이지 않는가

### 3.5 상태 피드백

- [ ] 관심 보내기/취소, 1촌 맺기, 대화 시작 등 **성공 시 즉시 메시지**
- [ ] 실패 시 한글로 원인 설명
- [ ] 네트워크 지연 시 로딩 상태 (스피너 또는 "처리 중...")
- [ ] 빈 상태에는 다음 행동 안내 (예: "사람 목록에서 관심을 보내보세요")

### 3.6 네비게이션

- [ ] 하단 탭 7개 이하 (현재 8개 → 리뷰 필요)
- [ ] 각 탭에 한글 라벨 (아이콘 only 금지)
- [ ] "뒤로" 버튼이 모든 상세 화면에 존재
- [ ] 홈 버튼으로 한 번에 홈 진입 가능

## 4. 흐름별 체크

### 회원가입 → 온보딩 → 홈
- [ ] 한 단계에 질문 3개 이하
- [ ] 필수 항목을 별표 또는 "(필수)" 명시
- [ ] 각 단계 이전/다음 버튼 가까이 배치
- [ ] 완료 후 "환영합니다" 또는 명확한 다음 안내

### 사람 탐색 → 관심 보내기
- [ ] 사람 카드 클릭 영역이 카드 전체
- [ ] 상세 진입 시 관계 액션이 하단 고정 또는 눈에 띄는 위치
- [ ] "관심 보내기" 성공 후 상태 즉시 변경

### 대화 시작
- [ ] "대화 시작" 가능 조건이 안내됨 ("서로 관심을 보내거나...")
- [ ] 대화 불가 시 왜 불가능한지 설명
- [ ] 대화 화면 진입 시 상대 닉네임이 헤더에 크게

### 모임 생성 → 참여
- [ ] 모임 생성 필수 항목 명확 (이름, 카테고리, 지역)
- [ ] 참여 버튼 누른 즉시 상태 변경
- [ ] 참여 후 모임 채팅 진입 경로 노출

## 5. 이번 작업에서 적용할 우선 개선 항목

코드 감사 기반 P0 수정 리스트:

1. **`text-xs` 사용 감사** — 35개소 중 메타데이터가 아닌 곳은 `text-sm` 이상으로 승격
2. **온보딩 페이지 인라인 스타일** — globals.css 기본 스타일이 안 먹는 `style={{}}` 구문 제거
3. **빈 상태 문구 통일** — [glossary §5](./glossary.md#5-안내-문구-표준-시니어-친화) 기준 적용
4. **에러 메시지 한글화** — supabase 원본 메시지 노출 구간 번역
5. **상태 변화 토스트** — 관심 보내기 성공 시 "관심을 보냈어요" 피드백
