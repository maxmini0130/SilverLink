# WithDay Design Guide

## 1. 문서 개요

- 문서명: WithDay Design Guide
- 서비스명: WithDay (위드데이)
- 문서 목적: 디자인 방향 정렬 및 바이브 코딩용 기준 문서
- 버전: v0.1
- 적용 범위: MVP ~ 초기 고도화

---

## 2. 디자인 한 줄 정의

**따뜻하고 정돈된 안심형 생활 관계 플랫폼**

위드데이는 시니어를 위한 서비스이지만, 단순히 글자를 크게 만드는 수준의 디자인이 아니라  
**믿을 수 있고, 편안하고, 품위 있으면서도 쉬운 사용성**을 갖춘 관계형 플랫폼으로 보여야 한다.

---

## 3. 디자인 목표

### 3.1 핵심 목표
1. 사용자가 서비스에 들어왔을 때 부담스럽지 않고 편안해야 한다
2. 관계 서비스이지만 가볍거나 자극적으로 보이면 안 된다
3. 시니어 친화 UX를 제공하되 촌스럽지 않아야 한다
4. 생활 피드, 사람 탐색, 모임, 대화가 하나의 통일된 경험으로 보여야 한다
5. 공개범위와 안전장치가 디자인에서도 느껴져야 한다

### 3.2 피해야 할 방향
- 복지앱처럼 지나치게 딱딱한 화면
- 소개팅앱처럼 과하게 핑크/하트 중심인 화면
- 20대용 SNS처럼 작고 복잡한 UI
- 과도한 댓글/숫자 경쟁 구조
- 시니어를 “디지털 약자”처럼 취급하는 과잉 배려 디자인

## 4. 브랜드 인상 키워드

### 4.1 핵심 키워드
- 따뜻한
- 정돈된
- 믿을 수 있는
- 편안한
- 생활감 있는
- 품위 있는
- 과하지 않은
- 관계를 이어주는

### 4.2 피해야 할 키워드
- 자극적인
- 유치한
- 형광색
- 촌스러운
- 너무 젊은
- 과하게 공공기관 같은
- 차갑고 병원 같은

## 5. 비주얼 컨셉

## 컨셉명
**Warm Trust Minimal**

### 컨셉 설명
위드데이는 미니홈피형 감성과 현대적 모바일 UI를 결합한 구조를 가진다.  
즉, 사람 냄새 나는 일상 공유 공간이면서도, 카드 중심의 정돈된 인터페이스를 유지해야 한다.

### 컨셉 원칙
- 카드 중심 구조
- 사진은 크게
- 버튼은 넓고 명확하게
- 배경은 부드럽게
- 정보는 세로 흐름으로 정리
- 시선이 자연스럽게 내려가도록 구성
- 과한 애니메이션보다 정적인 안정감 우선

## 7. 타이포그래피 가이드

### 7.1 기본 원칙
- 작은 글씨 금지
- 정보 계층은 분명하게
- 과한 Bold 사용 금지
- 줄간격을 충분히 준다
- 한 화면에 너무 많은 텍스트를 넣지 않는다

### 7.2 권장 사이즈
- Page Title: 24 ~ 28px
- Section Title: 20 ~ 22px
- Card Title: 18 ~ 20px
- Body: 17 ~ 18px
- Secondary Body: 15 ~ 16px
- Caption / Badge: 14 ~ 15px
- Button Text: 17 ~ 18px

### 7.3 권장 폰트
- Pretendard
- Noto Sans KR
- SUIT

### 7.4 금지 사항
- 본문 14px 이하
- 한 화면 안에서 폰트 스타일 남용
- 자간이 과하게 좁은 텍스트
- 지나치게 얇은 폰트 굵기

---

## 8. 레이아웃 가이드

### 8.1 기본 구조
- 상단 헤더
- 섹션 단위 본문
- 하단 탭 네비게이션
- 세로 스크롤 중심 구조

### 8.2 레이아웃 원칙
- 한 화면에 핵심 행동 1개를 강조
- 좌우 여백 충분히 확보
- 카드 간격을 넉넉히 둔다
- 리스트는 숨 막히지 않게 띄운다
- 탭은 5개 이내 유지

### 8.3 권장 수치
- Screen Padding: 16 ~ 20px
- Section Gap: 24 ~ 28px
- Card Gap: 12 ~ 16px
- Card Padding: 16 ~ 20px
- Button Height: 최소 48px / 권장 52 ~ 56px
- Radius: 16 ~ 24px

## 9. 컴포넌트 스타일 가이드

## 9.1 버튼
### Primary Button
- 메인 컬러 채움
- 흰색 텍스트
- 넉넉한 높이
- 둥근 모서리
- 가장 중요한 행동에만 사용

### Secondary Button
- 흰 배경 또는 아주 연한 배경
- 메인 컬러 테두리 또는 약한 그림자
- 덜 중요한 액션에 사용

### Tertiary / Text Button
- 최소 사용
- 시니어 UX에서는 버튼형 UI 우선

---

## 9.2 카드
### 공통 카드 원칙
- Surface는 흰색
- Radius는 16~24px
- 그림자는 얕게
- 내용은 세로로 자연스럽게 정리
- 카드 내부 정보가 너무 빽빽하지 않게

### 카드 종류
- Profile Card
- Feed Card
- Group Card
- Notification Card
- Empty State Card

---

## 9.3 배지(Badge)
### 사용 예
- 인증 완료
- 공개범위
- 모임 상태
- 1촌
- 관심 보냄

### 디자인 원칙
- 캡슐형
- 너무 작지 않게
- 설명 가능한 단어 사용
- 상태가 한눈에 보이게

예시:
- 전체 인증회원
- 1촌만
- 관심 있는 사람만
- 인증 완료
- 모집중

---

## 9.4 반응 칩(Reaction Chip)
위드데이는 댓글 대신 반응 중심 구조를 사용한다.

### 반응 문구 예시
- 좋아요
- 반가워요
- 취미가 비슷해요
- 같이 가보고 싶어요

### 디자인 원칙
- 둥근 pill 형태
- 과도한 색상 사용 금지
- 탭 가능한 크기 확보
- 숫자 경쟁 느낌을 주지 않게

---

## 10. 화면별 디자인 원칙

## 10.1 홈 화면
### 목적
사용자가 처음 들어왔을 때 무엇을 해야 하는지 보여주는 화면

### 우선순위
1. 오늘의 추천 사람
2. 내 주변 새 피드
3. 추천 모임
4. 알림 요약

### 디자인 포인트
- 기능 메뉴판처럼 보이면 안 됨
- 추천 중심 카드 UI여야 함
- 첫 행동이 명확해야 함
- 너무 많은 블록을 한 화면에 몰지 않기

---

## 10.2 사람 상세 프로필
### 목적
상대를 신뢰하고 관계를 시작할지 판단하는 화면

### 디자인 포인트
- 대표 사진은 충분히 크게
- 이름/지역/연령대는 선명하게
- 인증 상태를 신뢰감 있게 보여줌
- 관심사 태그는 읽기 쉽게
- 최근 공개 피드는 섬네일로 보여줌
- 주요 액션 버튼은 2~3개 이내

### 꼭 보여야 하는 요소
- 프로필 사진
- 이름/닉네임
- 지역/연령대
- 관심사
- 관계 목적
- 자기소개
- 최근 공개 피드
- 관심 / 1촌 / 대화 버튼

---

## 10.3 피드
### 목적
생활감과 분위기를 보여주고 자연스러운 대화 계기를 만드는 영역

### 디자인 포인트
- 사진은 크게
- 글은 짧고 편하게
- 댓글 없음
- 공개범위는 명확한 배지로
- 반응은 따뜻한 언어 사용
- 좋아요 수 경쟁 느낌은 줄인다

---

## 10.4 모임
### 목적
온라인 관계를 실제 활동으로 연결하는 영역

### 디자인 포인트
- 정보가 또렷해야 함
- 제목, 일정, 장소, 정원, 참여 인원이 잘 보여야 함
- 주최자 정보와 신뢰감 표시 필요
- 작은 예쁨보다 안심감이 우선

---

## 10.5 대화
### 목적
관계를 이어가되 편안하고 익숙한 경험 제공

### 디자인 포인트
- 말풍선 크기 충분히
- 입력창 크게
- 첫 메시지 가이드 칩 제공 가능
- 신고/차단 버튼 찾기 쉬움
- 지나치게 장난스럽지 않게

---

## 10.6 마이페이지
### 목적
내 미니홈피와 내 관계를 관리하는 개인 공간

### 디자인 포인트
- 내 프로필 요약이 먼저 보여야 함
- 내 피드는 사진 중심 그리드 가능
- 공개범위 관리 진입이 쉬워야 함
- 관계 현황(관심, 1촌 등)이 정리돼야 함

---

## 11. UX 원칙

### 11.1 시니어 UX 원칙
- 선택지를 줄인다
- 텍스트는 크게
- 버튼은 넓게
- 설명은 짧고 명확하게
- 숨겨진 기능보다 드러난 기능을 우선
- 아이콘만으로 의미 전달하지 말 것
- 항상 현재 상태가 보여야 함

### 11.2 상호작용 원칙
- 한번에 한 가지 핵심 행동만 강조
- 결과가 바로 보이게
- 성공/실패/제한 상태를 명확히 안내
- 관계 상태를 헷갈리지 않게 표현

---

## 12. 금지 패턴

### 하지 말아야 할 디자인
- 너무 작은 텍스트
- 너무 많은 아이콘 버튼
- 과도한 애니메이션
- 화려한 그라데이션 남발
- 소개팅앱 같은 하트 중심 화면
- 정보가 너무 빽빽한 리스트
- 댓글/숫자 경쟁 구조
- 스와이프 기반 평가형 UI

## 14. Tailwind 적용 가이드

### 추천 방향
- theme token을 먼저 정리
- 색상/spacing/radius/shadow를 재사용 가능한 형태로 관리
- 컴포넌트 클래스 중복을 줄이되 과한 추상화는 피함

### 예시 방향
- `bg-background`
- `text-foreground`
- `text-muted-foreground`
- `bg-primary`
- `text-primary-foreground`
- `rounded-2xl`
- `shadow-sm`

### 원칙
- MVP에서는 지나치게 복잡한 디자인 시스템 대신,
  공통 규칙을 정하고 핵심 컴포넌트에 우선 적용한다

## 15. 바이브 코딩 적용 원칙

AI에게 디자인을 맡길 때는 아래 기준을 함께 줘야 한다.

### 원칙
1. “예쁘게”가 아니라 “디자인 규칙”을 전달할 것
2. 한 번에 화면 하나씩 수정하게 할 것
3. 공통 토큰과 컴포넌트부터 만들게 할 것
4. 기존 기능 로직은 건드리지 말라고 명시할 것
5. 모바일 우선이라고 명확히 지시할 것
6. 시니어 UX를 유지하라고 반복적으로 알려줄 것

### 추천 순서
1. 디자인 토큰 정리
2. 공통 컴포넌트 정리
3. 공통 껍데기 적용
4. 홈 화면 적용
5. 사람 상세 적용
6. 피드 적용
7. 모임 적용
8. 대화 적용
9. 마이페이지 적용

---

## 16. 최종 한 줄 정리

위드데이의 디자인은
**“시니어를 배려하지만 올드하지 않고, 관계 서비스이지만 가볍지 않으며, 생활감 있는 사진과 카드 중심으로 편안하게 탐색되는 안심형 플랫폼 디자인”**
으로 구현해야 한다.
