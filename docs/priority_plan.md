# WithDay 우선순위 재정리표

## 1. 문서 개요

- 문서명: WithDay Priority Plan
- 목적: 현재 구현 상태를 기준으로 개발 우선순위를 재정렬하고, 기능 추가보다 안정화와 사용자 흐름 정리에 집중하기 위한 기준 문서
- 적용 범위: MVP 이후 1차 안정화 단계
- 기준일: 2026-04-15

---

## 2. 현재 단계 판단

현재 WithDay는 **기능이 부족한 상태가 아니라, MVP 핵심 기능이 상당 부분 구현된 상태**이다.

이미 구현된 주요 영역:
- 로그인 / 온보딩
- 홈
- 사람 상세
- 피드 목록 / 상세
- 마이페이지
- 관계(관심 / 1촌)
- DM
- 알림
- 모임 생성 / 상세 / 채팅
- 공개범위 RLS
- 신고 / 차단
- DB / Storage / Auth / RLS / RPC / Realtime 기반 구조

따라서 현재 단계의 핵심 과제는 다음과 같다.

1. 기능 추가보다 안정화가 우선
2. 핵심 사용자 흐름을 다시 정렬해야 함
3. 관계 모델을 단순하게 정리해야 함
4. 시니어 UX 검증이 필요함
5. 디자인은 기능 위에 얹는 것이 아니라, 흐름을 더 쉽게 만드는 방향으로 적용해야 함

---

## 3. 전체 우선순위 기준

현재 단계의 우선순위는 아래 순서로 본다.

1. 안정화
2. 핵심 사용자 흐름 정리
3. 관계 모델 단순화
4. 홈/문구/UX 정리
5. 디자인 시스템 정리
6. 신규 기능 추가

즉, 지금은 **“더 만들기”보다 “이미 만든 것을 사용자 관점에서 다듬는 단계”**이다.

## 4. 우선순위 재정리표

| 우선순위 | 영역 | 해야 할 일 | 목적 | 산출물 | 완료 기준 |
|---|---|---|---|---|---|
| P0 | 회귀 안정화 | 핵심 기능 전체 점검 | 이미 구현된 기능이 실제로 끊기지 않는지 확인 | 회귀 테스트 체크리스트 | 로그인~모임~대화까지 주요 흐름 모두 정상 |
| P0 | 핵심 사용자 흐름 | A/B/C 핵심 시나리오 끝까지 테스트 | 서비스 핵심 경험이 자연스럽게 이어지는지 확인 | 사용자 흐름 점검표 | 3개 흐름 모두 막힘 없이 수행 가능 |
| P0 | 관계 모델 | 관심 / 1촌 / 대화 관계 정의 확정 | 제품 개념 혼선을 줄임 | 관계 모델 정의서 | 팀 내 설명이 한 문장으로 통일됨 |
| P1 | 홈 화면 정렬 | 홈에서 무엇을 먼저 보여줄지 정리 | 첫 행동 유도 | 홈 정보 우선순위 문서 | 홈 진입 시 핵심 행동이 명확함 |
| P1 | 정책 문구 | “관심”, “1촌”, “대화 가능”, “공개범위” 등 용어 정리 | 사용자 혼란 감소 | 서비스 용어집 | 핵심 용어가 직관적으로 이해됨 |
| P1 | 시니어 UX | 글자, 버튼, 흐름, 설명 문구 점검 | 실제 사용 난이도 검증 | UX 개선 체크리스트 | 기본 태스크를 도움 없이 수행 가능 |
| P2 | 디자인 시스템 | 색상, 타이포, 카드, 버튼, 배지 정리 | 화면 통일감 확보 | 디자인 가이드 / 공통 컴포넌트 기준 | 주요 화면의 톤앤매너가 일관됨 |
| P2 | 화면 정리 | 홈 → 사람 상세 → 피드 → 모임 → 대화 순서로 리디자인 | 사용성 개선 | 화면별 개선안 | 핵심 화면에서 혼란 요소 감소 |
| P3 | 기능 고도화 | 추천 고도화, 결제, 보호자 기능 등 | 후속 확장 | 백로그 문서 | 현재 단계에서는 보류 유지 |

## 5. 지금 바로 해야 할 5가지

### 1) 기능 추가를 잠시 멈추고 핵심 사용자 흐름 점검
아래 3개 흐름이 막힘 없이 되는지 직접 테스트해야 한다.

#### 흐름 A
회원가입 → 온보딩 → 홈 → 사람 상세 → 관심 보내기 → 대화

#### 흐름 B
회원가입 → 온보딩 → 홈 → 피드 보기 → 반응 → 프로필 → 관계 연결

#### 흐름 C
회원가입 → 온보딩 → 홈 → 모임 → 참여 → 모임 채팅 → 후속 관계

### 2) “관심 / 1촌 / 대화” 관계 모델 정리
사용자 관점에서 아래 개념을 확정해야 한다.

- 관심: 가벼운 신호
- 1촌: 상호 승인된 관계
- 대화: 상호 관심, 1촌, 같은 모임 참여 시 가능

### 3) 홈 화면 우선순위 재정렬
홈은 기능 링크 모음이 아니라 첫 행동 유도 화면이어야 한다.

추천 우선순위:
1. 추천 사람
2. 최근 공개 피드
3. 추천 모임

### 4) 실제 회귀 테스트 시나리오 문서화
아래 항목은 체크리스트 형태로 관리해야 한다.

- 로그인 가능
- 온보딩 가능
- 프로필 수정 가능
- 피드 작성 가능
- 공개범위 적용 확인
- 관심 보내기 / 취소
- 1촌 맺기
- DM 시작
- 모임 생성 / 참여 / 나가기
- 알림 읽음
- 신고 / 차단

### 5) 정책 문구 정리
아래 용어를 사용자 친화적으로 다시 검토해야 한다.

- 관심 보내기
- 1촌 맺기
- 대화 가능
- 공개범위

## 7. 핵심 사용자 흐름 기준 우선순위

## 10. 주요 판단 포인트

현재 위드데이는 **“기능이 부족한 상태”가 아니라 “구조는 있는데 사용자 입장에서 더 정리해야 하는 상태”**이다.

### 지금 해야 하는 것
- 끊기지 않게 만들기
- 헷갈리지 않게 만들기
- 첫 행동이 보이게 만들기

### 지금 하면 안 되는 것
- 새로운 기능을 계속 붙이기
- 관계 모델을 더 복잡하게 만들기
- 디자인만 예쁘게 바꾸고 흐름을 놓치기

## 12. 최종 결론

현재 위드데이는 방향이 맞게 진행되고 있다.  
다만 이제는 구현 중심 단계에서 **안정화 + 사용자 흐름 정리 단계**로 넘어가야 한다.

따라서 지금의 핵심 우선순위는 다음과 같다.

1. 핵심 기능이 실제로 안정적으로 동작하는지 확인
2. 관심 / 1촌 / 대화 관계 구조를 사용자 입장에서 단순화
3. 홈에서 무엇을 먼저 해야 하는지 보이도록 정렬
4. 정책 문구와 UX를 시니어 관점에서 다듬기
5. 그 후 디자인 시스템을 적용하고 화면을 정리하기
