# SilverLink MVP — PRD (Product Requirements Document)

## 1) 제품 개요

- 제품명: SilverLink (가칭)
- 형태: 모바일 웹(PWA)
- 1차 목표: **관계 형성 루프**가 실제로 도는지 검증
  - **모임 참여 → 모임 채팅 → 하루 1회 기분 체크(재방문)**

## 2) 타깃 사용자

- 60~75세, 스마트폰 사용 가능(카톡/유튜브 등 사용), 지역 기반 취미·소통 니즈
- 파일럿 유입: 복지관/센터/지역 커뮤니티 리더를 통한 온보딩 가능

## 3) 문제 정의

- 시니어는 스마트폰을 갖고 있지만, “매일 들어갈 이유”와 “신뢰 가능한 관계 형성 채널”이 부족
- 기존 소개팅/커뮤니티 앱은 UI 복잡 + 사기 리스크 + 지역/오프라인 연결 약함

## 4) 핵심 가치 제안 (MVP)

1. **취미/지역 기반 모임**으로 부담 없이 연결
2. **모임 채팅**으로 관계 형성(1:1보다 안전/운영 효율 좋음)
3. **하루 1회 기분 체크**로 루틴/정서 데이터 축적(리텐션 훅)

## 5) 성공 지표(KPI)

### North Star Metric

- 주간 활성 사용자(WAU) 중 **“모임 참여 + 메시지 1회 이상”** 비율

### MVP KPI (최소 기준)

- Activation: 가입 후 24시간 내 `프로필완료 + 모임 상세조회` ≥ 60%
- Core Action (7일): `모임 참여` ≥ 30% AND `메시지 전송` ≥ 20%
- D7 Retention: 15~20% (오프라인 파일럿 기준 현실값)

## 6) MVP 범위

### 포함(In-Scope)

A. 인증/프로필

- 로그인: **이메일 매직링크(Passwordless)**
- 프로필: 닉네임, 나이대, 지역(구/동 단위), 취미(다중 선택)

B. 모임(코어)

- 모임 리스트(지역/카테고리 필터)
- 모임 상세
- 참여하기(가입)
- 내 모임 목록

C. 모임 채팅(코어)

- 모임 채팅방: 텍스트 메시지 송수신
- 멤버만 접근 가능(권한/RLS)

D. 리텐션 훅

- 기분 체크(1~5점) **하루 1회**
- 홈에서 “오늘 기분” 카드로 노출

E. 안전/운영(최소)

- 신고/차단 (2탭 이내)
- 운영정책: 금전요구/계좌/투자/외부메신저 유도 금지
- 관리자 최소: 신고 목록 확인 + 상태 변경(open/in_review/closed)

### 제외(Out-of-Scope, MVP 이후)

- 1:1 데이팅/스와이프
- 음성/사진/파일 전송
- 걸음수 자동 측정(센서/권한/정확도 이슈)
- OCR 복약 알림
- AI 추천 고도화(초기엔 룰 기반)

## 7) 사용자 여정(핵심 플로우)

### 온보딩 (첫 5분)

1. 로그인(이메일) → 자동 로그인
2. 프로필 작성(닉네임/나이대/지역/취미)
3. 홈에서 추천 모임 3개 노출
4. 모임 상세 → 참여하기 → 채팅방 입장

### 일상 루틴 (재방문)

- 홈 상단 “오늘 기분 체크” → 완료 시 칭찬 문구/작은 배지
- 내 모임 채팅 알림(추후 PWA 푸시 고려)

## 8) 정보구조(IA) / 화면 목록

- /login
- /onboarding
- / (Home)
- /groups (모임 리스트)
- /groups/[id] (모임 상세)
- /groups/[id]/chat (모임 채팅)
- /me (내 정보: 프로필 수정/내 모임/차단 목록)
- (관리자) /admin/reports (신고 목록)

## 9) UX 원칙(시니어 친화)

- 큰 글씨/큰 터치 영역(최소 44px 이상 권장)
- 한 화면 1목표(과한 정보 밀도 금지)
- 네비게이션 단순(하단 4탭 권장)
- 빈 상태/로딩/에러 메시지 명확히
- “연애앱” 느낌 최소화: **취미/모임 우선 → 관계 자연 확장**

## 10) 운영 정책(최소)

- 금전 요구/계좌/투자 권유/다단계/외부 링크 유도 금지
- 신고 누적/패턴(전화번호/카톡ID 등) 시 운영자 검토
- 개인정보 최소 수집, 민감정보(정서 로그) 목적 명시

## 11) 릴리즈 계획(권장)

- Sprint 0(2~3일): 프로젝트/인증/DB 스키마
- Sprint 1(1~2주): 프로필/모임 리스트·상세
- Sprint 2(1~2주): 참여/채팅(Realtime) + 권한(RLS)
- Sprint 3(1주): 기분 체크 + 신고/관리자 최소 + 이벤트 로깅
- 파일럿 운영: 200~500명 목표

## 12) MVP 완료(Definition of Done)

- 신규 사용자 5분 내 `모임 참여 + 채팅방 입장` 가능
- 기분 체크(일 1회) 저장/표시 정상
- 모임 채팅은 **멤버만** 읽기/쓰기 가능
- 신고/차단 동작 + 관리자 화면에서 신고 조회 가능
- 핵심 이벤트 로깅 가능(아래 이벤트 목록)
